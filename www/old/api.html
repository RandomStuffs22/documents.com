GET /v1/documents
			Gets JSON of document ids, names, and versions

		POST /v1/documents
			Create new document with name and content parameters (application/x-www-form-urlencoded)

		GET /v1/documents/document_id
			Gets JSON representation of full document.

		PUT /v1/documents/document_id
			Update document name and content. Must also pass in version of the document that is getting updated
			to deal with conflicts. (application/x-www-form-urlencoded)

		DELETE /v1/documents/document_id?version=version
			Delete document and all associated edits. Version query parameter is required and must match the documents current
			version for the delete to be successful.

		GET /v1/documents/document_id/edits?start=0&end=2
			Get document edits (in patch format) that took place from start and end version parameters.

		POST /v1/documents/document_id/edits
			Post new document edit (application/x-www-form-urlencoded). The edit can include a new name parameter, a patch parameter, or both.
