GET /v1/documents
			Gets JSON list of existing document ids, names, and versions

		POST /v1/documents
			Create new document with "name" and "content" parameters json encoded in post body. Response body includes recorded name, content and documents assigned ID.

		GET /v1/documents/document_id
			Gets JSON representation of full document. Attributes include "name", "content", and "version"

		PUT /v1/documents/document_id
			Include at least one of "name" or "content", also must include "version". Encode these as json key value table in PUT body.
			The version should be the clients version of the document. PUT is implemented as follows on the server:
				- server does a diff between PUT content and the document content for "version" that is stored on server.
				- server then applies that diff to 
							
			Update document name and content. Must also pass in version of the document that is getting updated
			to deal with conflicts. (application/x-www-form-urlencoded)

		DELETE /v1/documents/document_id?version=version
			Delete document. Version query parameter is required and must match the documents current
			version for the delete to be successful.

		GET /v1/documents/document_id/edits?start=0&end=2
			Get document edits (in patch format) that took place from start and end version parameters.

		POST /v1/documents/document_id/edits
			Post new document edit (application/x-www-form-urlencoded). The edit can include a new name parameter, a patch parameter, or both.
